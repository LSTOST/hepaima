# 我们合拍吗 - 首页设计文档

## 📋 项目概述

### 产品名称
**我们合拍吗**（域名：hepaima.com）

### 产品定位
一款基于心理学理论的**情侣契合度测评工具**，帮助情侣更科学地了解彼此，改善关系质量。

### 核心价值主张
> "用科学的方式，读懂你们的爱情密码"

### 产品背景
- 基于依恋理论（Attachment Theory）、爱的语言（5 Love Languages）、Gottman 婚姻研究等经典心理学模型
- 提供三个阶段的测评：暧昧期、热恋期、稳定期
- 双人测评模式：A完成后邀请B，双方都完成后生成配对报告
- 付费报告：基础版免费，标准版/深度版付费解锁更多分析

## 👥 目标用户

### 用户画像

| 维度 | 描述 |
|------|------|
| **年龄** | 18-35岁 |
| **性别** | 男女均衡，女性略多（预估6:4） |
| **关系状态** | 暧昧中、恋爱中、同居、已婚 |
| **教育背景** | 大专及以上为主 |
| **地域** | 一二线城市为主 |
| **消费能力** | 中等，愿意为情感价值付费 |

### 用户场景

1. **暧昧期用户**
   - 心理：不确定对方是否合适，想要一个"科学依据"
   - 需求：快速、低门槛地测试兼容性
   - 顾虑：不想让对方觉得太正式/太快

2. **热恋期用户**
   - 心理：好奇彼此的契合度，想更了解对方
   - 需求：有趣的互动活动，可以一起讨论结果
   - 顾虑：希望结果是积极的

3. **稳定期用户**
   - 心理：关系遇到瓶颈，或想让关系更好
   - 需求：深度分析，可执行的改善建议
   - 顾虑：价格是否值得，报告是否真的有用

### 用户痛点

- 不知道如何更好地了解伴侣
- 沟通中经常产生误解
- 想改善关系但不知道从何入手
- 市面上的测试要么太娱乐化，要么太学术化

## 🎯 首页目标

### 主要目标（按优先级）

1. **快速传达产品价值** - 用户在5秒内理解这是什么产品
2. **引导用户开始测评** - 清晰的行动路径，减少决策成本
3. **建立信任感** - 体现专业性和可靠性
4. **激发情感共鸣** - 让用户觉得"这就是我需要的"

### 关键指标

- 首屏到阶段选择的点击率
- 整体跳出率
- 测评开始转化率

## 🏗️ 页面结构

```
首页

│   ├── 暧昧期卡片
│   ├── 热恋期卡片（推荐标记）
│   └── 稳定期卡片
│
├── 信任背书区
│   ├── 科学依据说明
│   └── 测评维度介绍
│
├── 报告预览区
│   └── 报告内容示意

│   └── 再次引导开始测评

    └── 版权/隐私政策
```

### 各区块详细说明

#### 1. 导航栏
- **Logo**：品牌名"我们合拍吗"，可考虑简单图形标识
- **邀请码入口**：文字链接，点击弹出输入框
- **历史记录**：按钮样式，查看过往测评

#### 2. Hero区域
- **主标题**：`我们合拍吗？`
- **副标题**：`用科学的方式，读懂你们的爱情密码`
- **社会证明**：
  - 已有 30,000+ 对情侣完成测评
  - 平均契合度 78.5%
  - 用户好评率 96%
- **主CTA**：引导滚动到阶段选择区

#### 3. 阶段选择区（最重要）
三个阶段卡片，每个包含：

| 阶段 | 暧昧期 | 热恋期 | 稳定期 |
|------|--------|--------|--------|
| **副标题** | 还在互相了解 | 确定关系中 | 1年+/同居/已婚 |
| **题目数** | 25题 | 32题 | 40题 |
| **时长** | 约5分钟 | 约7分钟 | 约10分钟 |
| **起步价** | ¥9.9 | ¥14.9 | ¥19.9 |
| **标记** | - | 最多人选 | - |

**设计要点**：
- 热恋期需要视觉突出（推荐标记）
- 卡片可点击，hover有反馈
- 价格不是重点，但需要展示

#### 4. 信任背书区
- **科学依据**：
  - 依恋理论 - 了解你们如何建立亲密关系
  - 爱的语言 - 发现表达和接收爱的方式
  - Gottman研究 - 预测关系走向的关键指标
- **可选**：简单说明测评维度

#### 5. 报告预览区
展示报告包含的内容：
1. 契合度总览
2. 依恋类型配对分析
3. 爱的语言匹配
4. 成长建议与情侣任务

**设计要点**：
- 可以用简化的UI示意图
- 不需要完整展示，制造期待感

#### 6. 底部CTA
- 简短文案 + 按钮
- 例：`准备好更了解彼此了吗？` + `开始测评`

## 🎨 设计方向

### 整体调性

| 关键词 | 说明 |
|--------|------|
| **温暖** | 让人感到亲近、舒适，不冰冷 |
| **专业** | 有可信度，不是娱乐测试 |
| **现代** | 符合年轻人审美，不老旧 |
| **精致** | 细节讲究，有品质感 |
| **克制** | 不花哨，信息清晰 |

### 避免的风格

❌ 太多emoji和卡通元素（显得幼稚）
❌ 过于极简冷淡（缺乏情感温度）
❌ 大量渐变和霓虹色（显得廉价）
❌ 过度装饰（干扰信息传达）

### 配色建议

**主色调**：温暖的粉紫色系
- 可以参考：#EC4899（粉）、#8B5CF6（紫）
- 用于强调元素、CTA按钮

**辅助色**：
- 中性灰用于正文和次要信息
- 浅色背景保持页面透气感

**示例色板**：
```
主色：#EC4899 或 #8B5CF6
强调：渐变 #EC4899 → #8B5CF6
背景：#FFFFFF / #FAFAFA / #FDF2F8
正文：#1F2937
次要文字：#6B7280
边框：#E5E7EB
```

### 字体建议

- 中文：PingFang SC / Noto Sans SC
- 英文/数字：SF Pro / Inter
- 字重：标题用 Semibold/Bold，正文用 Regular

### 参考案例

**国内产品**：
- 小红书 - 温暖、年轻、有品质感
- 即刻 - 简洁、现代、社区感
- 小宇宙 - 精致、克制、有调性

**国际产品**：
- Notion - 简洁专业
- Linear - 现代精致
- Headspace - 温暖疗愈

## 📱 响应式要求

### 移动端（主要）

- 预计 80%+ 流量来自移动端
- 首屏必须包含：标题 + 社会证明 + 可见的阶段选择入口
- 阶段卡片可左右滑动或垂直堆叠
- 按钮大小适合手指点击（最小44px高度）

### 桌面端

- 最大内容宽度：1000-1200px
- 阶段卡片可三列并排
- 保持视觉平衡，不要过于空旷

## 🔧 功能说明

### 交互元素

1. **阶段卡片**
   - 点击 → 跳转到 `/quiz?stage=xxx`
   - Hover → 视觉反馈（阴影/边框变化）

2. **邀请码入口**
   - 点击 → 弹出模态框
   - 输入6位邀请码 → 跳转到 `/quiz?invite_code=xxx`

3. **历史记录**
   - 点击 → 跳转到 `/history`

4. **主CTA按钮**
   - 点击 → 平滑滚动到阶段选择区

社会证明数据（可后期改为动态）：

- 好评率：96%

## 💬 沟通备注

### 当前问题

之前的首页设计存在以下问题：
1. 过多emoji，显得幼稚不专业
2. 或者过于冷淡极简，缺乏情感温度
3. 阶段卡片留白过多，信息密度低
4. 视觉层次不清晰，重点不突出

### 期望改进

1. 在温暖和专业之间找到平衡
2. 首屏有视觉冲击力，能抓住注意力
3. 阶段选择区是核心，需要突出但不花哨
4. 整体有品质感，符合目标用户审美
